    Imply Druid를 활용한 실시간 데이터 분석과 당근마켓 사례

    2024, Jun 16

    세션 정보

    • 장소: 1층 그랜드볼룸 105호
    • 시간: 4:30 ~ 5:00

    세션 목차

    • 이벤트 데이터 분석의 요구사항
    • Imply Druid를 활용한 실시간 집계 분석
    • 당근의 실시간 광고성과 분석 사례
    • Imply Druid 시작하기

    세션 내용

    Summary

    • 이벤트 데이터 분석의 요구사항
      • Apache Kafka, Amazon Kinesis, Apache Pulsar를 사용해 이벤트 데이터 기반 스트리밍 플랫폼을 구성함
      • 시간, 장소, 인력, 자재, 비용 등 여러 관점에서 데이터를 종합적으로 분석하고 통합된 실시간 및 과거 데이터 분석이 필요함
      • 실시간 분석으로 알림 제공 및 신속한 모니터링 기능을 갖추는 것이 필요함
    • Imply Druid를 활용한 실시간 집계 분석
      • TB-PB 크기의 데이터에 대해 1초 미만으로 쿼리하고 초당 100-1000개 쿼리를 처리할 수 있음
      • Kafka, Kinesis의 스트림 데이터를 실시간으로 수집 및 처리하며 내결함성과 지속적 백업을 통해 무중단 안정성을 제공함
      • Apache Druid는 대규모 데이터 분석 및 실시간 데이터 처리 애플리케이션의 요구사항을 수용하도록 설계되었음
    • 당근의 실시간 광고성과 분석 사례
      • 당근은 광고업자와 마케터를 위한 전문가 모드 플랫폼을 통해 실시간 성과 조회 기능을 제공함
      • 지역, 사용 OS, 연령, 성별 등 다양한 조건의 다차원 실시간 성과 조회 기능이 필요함
      • Imply Druid의 지원으로 빠른 도입과 운영 효율화를 통해 실시간 성과 데이터를 약 7만 명의 사용자에게 제공함
    • Imply Druid 시작하기
      • Imply Druid는 Apache Druid를 완성하는 플랫폼으로 빠른 적용과 운영에 대한 지원을 받을 수 있음
      • 여러 Native 서비스에 바로 통합이 가능하며 AWS와 통합 운영 및 결제를 제공함
      • 설계 단계 참여와 운영 단계에서의 최적화를 지원하여 대용량 데이터의 실시간 처리를 가능하게 함

    이벤트 데이터 분석의 요구사항

    • 이벤트 데이터 기반 메시지를 사용할 때 `Apache kafka, Amazon Kinesis, Apache Pulsar를 사용해서 스트리밍 플랫폼을 구성

    [표준화된 메시지 시스템]

    • 시간, 장소, 인력, 자재, 비용 등 여러 관점에서 데이터를 종합적으로 분석하는 것이 필요
    • 실시간과 과거 데이터를 함께 통합하여 분석하는 것이 필요
    • 데이터를 실시간으로 분석해서 알림을 제공하고, 여러 시스템과 연계해서 신속하게 모니터링할 수 있는 기능이 필요

    [데이터에 대한 분석 요구사항]

    • 스트리밍 데이터를 배치 형태로 전환하면 지연 시간이 증가하게 되고 실시간으로 인사이트를 얻을 수 없어 생산성이 저하됨
    • 실시간 분석을 위해 스트리밍 데이터 형태를 그대로 처리해야 함

    [스트리밍 데이터의 실시간 분석]

    • Druid는 오픈소스로 공개된 Apache의 Druid와 공식 지원을 받을 수 있는 Imply의 Druid가 있음
    • Druid는 실시간 데이터를 분석할 수 있게 스트리밍 그대로 처리하도록 설계됨
    • 별도 커넥터 없이 직접 연결됨
    • Druid의 특징
      • 이벤트 기반 데이터 입수
      • 확장성, 정합성을 가짐
      • 과거 + 실시간 데이터 조회
      • 자동화된 백업
      • 스키마 자동 탐지

    [Druid의 특징]

    Imply Druid를 활용한 실시간 집계 분석

    • 분석 애플리케이션은 TB-PB 크기의 데이터에 대해서도 1초 미만으로 쿼리를 할 수 있음
    • 고효율 엔진으로 초당 100-1000개 쿼리를 처리할 수 있어, 최저 비용으로 높은 동시성을 가짐
    • Kafka, Kinesis의 스트림 데이터를 실시간으로 수집하고 처리할 수 있어, 스트림의 이점을 살릴 수 있음
    • 내결함성과 지속적 백업을 통해 무중단 안정성이 있음

    [분석 애플리케이션의 필요성]

    • Apache Druid는 대규모 데이터의 분석뿐만 아니라, 실시간 데이터 처리가 필요한 애플리케이션의 요구사항도 수용할 수 있도록 설계되었음

    [Druid에 대한 설명]

    • 데이터 웨어하우스(분석)와 트랜잭션 데이터베이스(애플리케이션) 관점에서 각각 아래와 같은 역할을 수행할 수 있음
      • 최적화된 읽기 성능, 실시간 분석, 1초 미만의 응답 속도, 동시성 처리, 실시간 데이터 처리
    • 아래와 같은 시스템을 구성할 때, 유용하게 활용할 수 있음
      • 실시간 광고 분석 플랫폼
        • 광고 클릭 데이터를 실시간으로 수집하고 분석하여, 즉각적인 광고 전략 조정을 가능하게 함
        • BI 보고서 작성과 같은 분석 작업뿐만 아니라, 실시간으로 데이터를 처리하고 사용자에게 피드백을 주는 애플리케이션의 역할 수행
      • IoT 데이터 처리 시스템
        • 다양한 IoT 센서에서 실시간으로 데이터를 수집하고 분석
        • 이상 징후를 빠르게 탐지하고 경고를 발생시키는 애플리케이션의 역할 수행

    [Druid의 구제적인 특징]

    • Kafka에서 Partition을 확장하면, 그에 맞춰 Druid의 전달 태스크도 확장이 가능

    [Kafka와 Druid의 확장에 대한 도식]

    • 넷플릭스에서 로그 메트릭을 효율적으로 처리하기 위해 Druid를 활용
    • 초당 2백만개의 이벤트를 처리하는 높은 처리량
    • 1초 미만의 시간으로 1.5조개의 row를 검색이 가능한 높은 DB 성능
    • 데이터를 미리 집계하고 인덱싱하여 요약 저장함으로 row의 개수를 2배 줄이는 효과적인 필터, 압축이 가능

    [Druid를 사용한 넷플릭스 예시]

    • 컨플루언트에서 여러 Container의 대규모 로그를 관리하기 위해 Telemetry의 파이프라인에 Druid를 활용
    • 높은 이벤트 처리량, 높은 읽기 성능

    [Druid를 사용한 컨플루언트 예시]

    • 레딧에서 로그 메트릭을 효율적으로 처리하기 위해 Druid를 활용
    • 높은 이벤트 처리량, 높은 읽기 성능, 고가용성

    [Druid를 사용한 레딧 예시]

    당근의 실시간 광고성과 분석 사례

    • 당근에는 광고업자, 마케터 등 전문가를 위한 전문가 모드 플랫폼을 오픈함

    [당근의 전문가 모드 플랫폼]

    • 여러 조건의 조합으로 분석할 수 있는 다차원 실시간 성과 조회가 필요

    [전문가모드 플랫폼의 요구사항]

    • 다차원 실시간 성과라는건 지역 + 사용 OS + 연령, 성별 등의 조건들이 있고, 이를 실시간으로 조회하는 기능임

    [다차원 실시간 성과 조회 예시]

    • Druid 생태계에서 가장 앞서 이끄는 회사로, 빠르고 정확한 B2B 지원이 가능
    • 빠른 도입을 위해, 스키마와 파이프라인 설계 단계에서 함께하며, 비즈니스 요구사항의 PoC(Proof of Concept)까지 지원해줌
    • 주기적인 리뷰를 통해 쿼리 최적화와 운영에 필요한 옵션을 제안

    [Imply의 Druid를 선택한 이유]

    • Druid 이전에는 Batch 형태로 스트리밍 데이터를 처리해서, 실시간 요구사항을 충족하기 어려웠음

    [Druid 이전의 데이터 파이프라인]

    • 배치로 사전 정의한 집계 방식에 대해서만 조회 가능 (실시간X)
    • 실시간 다차원 보고서와 같은 고도화된 광고 플랫폼 기능을 제공하기 어려웠음

    [이전 데이터 파이프라인의 문제]

    • Druid를 적용하면서 실시간 분석을 할 수 있는 API를 제공할 수 있게 됨

    [Druid를 적용한 데이터 파이프라인]

    • 현재 대규모 데이터와 데이터 셋을 문제없이 운영 중
    • 쿼리 요청 중 95% 정도는 300ms 이하로 걸리는 퍼포먼스를 얻음
    • 약 7만명의 사용자에게 실시간 성과 데이터를 문제없이 제공 중

    [Druid 적용 후 얻은 이점]

    • Druid에서 지원하는 데이터 구조로 스키마 변경이 어렵거나 데이터 형태가 자주 변경될 때 유용
      • e.g. A/B 테스트 정보

    [Druid의 Nested columns]

    • SQL Query 문법으로 외부 데이터 전달 가능
    • 기존 data를 활용해서 중복 제거하고 전달도 가능
    • 쿼리 처리를 담당하는 워커의 개수를 지정하는 것이 가능

    [Druid의 Multi Stage Query Engine]

    • 오래 사용하지 않은 Data까지 쿼리하지만, 쿼리에 대해 높은 성능을 보여줌
    • 쿼리 워커가 병렬로 처리한 후, 결과를 외부에 저장하는 것이 가능
    • 대용량 보고서 등의 다운로드에 적합함

    [Druid의 Query From Deep Storage]

    • 여러 조건으로 이루어진 다차원 쿼리하지만 1초 내에 조회하는 높은 쿼리 성능을 보여줌
    • 다차원 쿼리의 결과를 효과적으로 전달하는 UI/UX를 제공

    [Imply Druid에서 제공하는 PIVOT(Query UI)으로 조회한 당근의 데이터 사용]

    • 광고 플랫폼의 실시간 성과 분석에 최적화되어 있는 솔루션
      • 다양한 차원과 기간에 대해, 약 200만 건의 데이터를 약 300ms 내로 조회함
    • Imply의 Druid를 사용할 경우, Druid 전문가들에게 지원을 받아 비용/시간을 절감할 수 있음
      • Imply의 Druid는 이미 해외 유명 기업들의 적용, 최적화 사례를 통해 검증됨

    [당근 사례의 요약]

    Imply Druid 시작하기

    • Imply Druid는 Apache Druid를 완성하는 플랫폼으로, 빠른 적용과 운영에 지원을 받을 수 있음

    [Imply Druid에 대한 이점]

    • 여러 Native 서비스에 바로 통합이 가능
    • AWS를 통한 통합 운영 및 결제를 제공
    • 인증받은 AWS의 파트너사임

    [AWS와 긴밀하게 통합이 가능]

    • 빠른 적용을 위한 설계 단계 참여와 운영 단계에서 최적화까지 지원받을 수 있음

    [Imply의 이점 소개]

    대용량 데이터를 다차원으로 핸들링하는 쿼리, 대용량 다운로드 등 배치를 활용해서 생산성이 떨어지는 작업들, 실시간 데이터를 빠르게 핸들링해야 하는 문제를 해결한 사례
