    세션 내용

    Summary

    • 들어가며
      • 과거에는 데이터를 찾기 위해 많은 시간을 도서관에서 보냈음. 이제는 데이터를 디지털화하고 머신러닝(ML)을 통해 신속하게 활용할 수 있음. 기존 ML 모델은 데이터를 레이블링하고 학습하여 배포하는 단계를 거쳤으나, 파운데이션 모델은 비가공 데이터를 준비하여 파인 튜닝 후 활용하는 단계를 거침.
    • Gen AI 앱 구축: 새로운 기술 스택
      • Amazon SageMaker는 여러 파운데이션 모델을 제공하는 ML 허브로, 모델 배포, 파인 튜닝, 컴퓨팅 인스턴스, 임베딩 관리가 가능함. SageMaker JumpStart는 모델 선택, 파인 튜닝, 테스트, 배포 기능을 자동화하여 제공함. Amazon Kendra는 사용자의 데이터를 연동하고 메타데이터를 수집하여 권한에 맞는 전문적인 답변을 생성하는 지능형 검색 서비스임.
    • Demo 1: Amazon SageMaker 및 Amazon Kendra를 사용한 RAG
      • SageMaker의 모델에 질문할 때 Kendra를 이용해 RAG를 구현할 수 있음. Kendra는 다양한 문서 데이터에서 검색을 수행하고 이를 바탕으로 답변을 생성함. llama3에 RAG를 적용한 데모에서는 Kendra를 통해 할루시네이션 문제를 해결하고, LLM이 Kendra의 검색 정보를 바탕으로 프롬프트를 구성하여 응답함.
    • Demo 2: Amazon Bedrock과 벡터 데이터베이스를 사용한 RAG
      • RAG를 위한 데이터 플랫폼으로 벡터 DB, 데이터 웨어하우스(DW), 데이터 레이크 등이 있음. Bedrock은 제공하는 API를 사용하여 원하는 모델을 만들고 접근, 파인 튜닝이 가능한 서버리스 서비스임. Bedrock은 모델 선택, 파인 튜닝, RAG 설정, 사용의 4단계로 간단하게 모델을 생성할 수 있으며, 플레이그라운드 기능을 통해 실험도 가능함.
    • 마무리
      • SageMaker와 Kendra를 활용하거나 Bedrock과 벡터 데이터베이스를 활용하여 RAG 기반의 LLM 서비스를 구축할 수 있음. 파운데이션 모델을 활용할 때 고려해야 할 핵심 사항들이 있으며, AWS AI 서비스 관련 참고 자료도 제공됨.

    들어가며

    • 이전에는 필요한 데이터를 찾기 위해 많은 시간을 사용했었음
    • 이제는 필요한 데이터를 디지털 데이터로 전환하고, ML을 통해 많은 시간을 들이지 않고 활용할 수 있음

    [필요한 정보를 도서관에서 찾아야 했음]

    • 기존 전통적인 ML모델은 데이터에 레이블을 지정 > 학습해서 모델 생성 > 배포 > 사용 단계를 거쳤음
    • 파운데이션 모델은 가공하지 않은 데이터 준비 > 파운데이션 모델을 파인 튜닝해서 모델 조정 > 활용 단계를 거침

    [기존과 현재 모델 사용 방식]

    • 역할을 부여(명령) > 상황 설명(맥락) > 추론하고자 하는 내용 입력(질문) 과정을 거쳐, 답변을 받는 시나리오
    • 서비스와 모델, 인프라의 관계
      • 애플리케이션에서 요청 > 모델에서 추론 후 반환
        • 자체 생성 파운데이션 모델, 공유되어 있는 파운데이션 모델, 모델 허브 등을 사용할 수 있음
      • 모델은 ML 플랫폼, 하드웨어 인프라를 통해 훈련되고 추론, 배포됨

    [파운데이션 모델을 사용한 예시]

    [할루시네이션 문제가 있음]

    • 커스터마이징은 프롬프트 엔지니어링, 파인 튜닝, 검색 증강 생성(RAG) 방식이 있음

    [파운데이션 모델의 커스터마이징 방법 3가지]

    Gen AI 앱 구축: 새로운 기술 스택

    [기존 도식에서 RAG를 위해 오케스트레이션을 추가]

    • Amazon SageMaker는 여러 파운데이션 모델을 제공하는 ML허브임
    • 모델 배포, 파인 튜닝, 컴퓨팅 인스턴스, 임베딩을 위한 엔드포인트까지 관리가 가능

    [Amazon SageMaker에 대한 설명]

    • SageMaker JumpStart는 모델 선택, 파인 튜닝, 테스트, 배포 기능을 제공하고 이를 자동화하는 서비스까지 제공

    [SageMaker JumpStart에 대한 설명]

    • Amazon Kendra는 자연어 질의를 통해 기존보다 나은 정보 검색을 할 수 있도록 한 지능형 검색 서비스임
    • 사용자의 데이터를 연동하고 메타데이터를 수집하고, 권한에 적합한 정보들을 사용하여 내가 가진 권한에 맞는 전문적인 답변을 생성

    [Amazon Kendra에 대한 설명]

    • Q Developer는 python을 포함한 여러 언어, 여러 IDE를 지원
    • 오픈소스 사용 시 고려할 수 있도록 라이센스(MIT, Apache, …) 종류를 안내 가능

    [Amazon Q Developer 사용]

    Demo 1: Amazon SageMaker 및 Amazon Kendra를 사용한 RAG

    • SageMaker에 있는 모델에 질문을 할 때, Kendra를 이용해서 RAG를 구현할 수 있음
    • Kendra를 이용해서 비정형, 반정형, 정형의 다양한 문서 데이터에서 검색을 하고 답변 생성에 활용할 수 있음

    [Kendra를 이용한 RAG 구현]

    • Notebook with Amazon Q Developer, SageMaker, Kendra를 활용해서 llama3에 rag를 적용한 데모 시연
    • LLM 만을 이용했을 때는 할루시네이션이 발생하는 문제가 있었음
    • Kendra를 통해 RAG 접근 구조로 만들어서 문제를 해결
    • LLM은 Kendra 검색 정보를 바탕으로 프롬프트를 이용해서 응답을 구성

    [RAG 방식의 서비스에 질문하는 데모 시연]

    Demo 2: Amazon Bedrock과 벡터 데이터베이스를 사용한 RAG

    • RAG를 위한 데이터 플랫폼으로 벡터 DB, Data Warehouse(DW), 데이터 레이크 등이 있음
    • ML 플랫폼을 위한 도구로 개발자 도구, Role and Privilege 정책 관리, QA, 모니터링, 플러그인 등이 있음

    [데이터와 도구에 대한 설명]

    • Bedrock은 제공하는 API를 사용해서 원하는 모델을 만들고 접근, 파인튜닝 하는 것이 가능한 서버리스 서비스임

    [파운데이션 모델을 다루기 위한 두 개의 서비스 비교]

    • 모델 선택, 파인 튜닝, RAG 설정, 사용으로 4단계만으로 간단하게 생성 가능

    [Bedrock의 모델 사용 과정]

    • Bedrock은 플레이그라운드 기능으로 원하는 모델에 대한 실험을 할 수 있음

    [Bedrock의 텍스트 플레이 그라운드로 Claude3 Sonnet 모델 테스트 과정 시연]

    • Boto3라는 Python AWS SDK
      • Boto는 아마존 강에 서식하는 돌고래 종류에 부여된 포르투갈어 이름에서 유래된 것으로, 아마존에서 헤엄친다고 해서 붙여진 이름

    [SageMaker의 노트북 기능으로 작성한 Bedrock API 사용 코드]

    • Pinecone은 빌리언 단위의 벡터 형태 아이템을 수 밀리초 만에 검색할 수 있는 데이터베이스

    [Pinecone을 이용한 인덱스 생성 및 확인]

    • LangChain으로 벡터 DB를 활용한 rag 적용한 흐름

    • SageMaker + Kendra 또는 Bedrock + 벡터 데이터베이스를 활용해서 RAG를 활용한 LLM 서비스 구축이 가능
