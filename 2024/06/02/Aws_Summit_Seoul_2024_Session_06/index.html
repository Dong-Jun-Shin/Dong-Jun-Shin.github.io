    Summary

    • 소프트웨어 개발 라이프사이클에 존재하는 기존의 어려움들
      • 개발자들은 대부분의 시간을 코드 작성 이외의 부수적인 운영 업무에 소비하여 혁신과 변화에 투자할 시간이 부족
      • 개발주기 내 다양한 업무로 인해 실제 코드 작성에 할애할 시간이 줄어듦
      • 운영 업무에 많은 시간을 소비하게 되어 개발 생산성이 저하
    • AWS Q Developer - 소프트웨어 개발을 위한 가장 강력한 생성형 AI 기반 어시스턴트
      • AWS Q는 보안과 개인 정보 보호를 전제로 한 생성형 AI 기반 어시스턴트로, 개발주기의 모든 단계에서 지원을 제공
      • 계획, 코드 작성, 테스트, 보안 검사, 디버깅, 최적화 등 다양한 기능을 제공하여 업무 생산성을 향상
      • 콘솔, IDE, 모바일 앱, Slack 등 다양한 매체에서 사용할 수 있으며, 자연어를 사용하여 통합 질문 및 솔루션 제안이 가능
    • 데모: 방치된 Java 애플리케이션 현대화하기
      • 현재 버전 지원 종료, 담당자 부재, Java 개발 경험 부족 등 다양한 문제 상황을 가정한 데모 시나리오
      • AWS Q Chat과 Business 기능을 통해 필요한 시스템 상황 조사 및 버전 업그레이드 방법을 파악
      • AWS Q Developer Agent를 통해 Java 버전 업그레이드를 진행하고, 코드 주석 추가, 테스트 커버리지 향상 및 Lambda 문제 해결까지 한 후 운영에 필요한 예상 비용을 분석하는 시나리오

    소프트웨어 개발 라이프사이클에 존재하는 기존의 어려움들

    • 부수적인 운영 업무로 인해 혁신과 변화에 투자할 수 있는 시간이 너무 적음

    [업무 시간 사용 비율에 대한 통계]

    • 개발자는 개발주기 중 코드 작성 이외의 업무에 대부분의 시간을 보냄

    [개발주기별 필요한 업무]

    AWS Q Developer - 소프트웨어 개발을 위한 가장 강력한 생성형 AI 기반 어시스턴트

    • AWS Q는 생성형 AI 기반의 어시스턴트로 Developer와 Business가 있음
    • 보안과 개인 정보 보호를 전제로 구현된 서비스이기 때문에 안전한 사용이 가능

    [AWS Q 소개]

    • 생성형 AI를 활용할 떄 맥락에 대한 이해 부족, 보안, 데이터 프라이버시, 회사 규정 문제가 있음

    [직장에서 생성형 AI를 활용할 떄 문제점들]

    • AWS Q Business는 기존 데이터를 바탕으로 맥락을 이해할 수 있음

    [AWS Q Business 소개]

    • 사용자의 권한을 확인하고 알맞은 데이터를 바탕으로 통합 검색 환경과 컨텐츠 생성 기능을 제공함

    [AWS Q Business의 기능]

    • AWS Q Developer는 설계, 구현, 테스트, 운영 등 소프트웨어 개발주기 전 과정에 걸쳐 어시스턴트 기능을 제공하고 업무 생산성을 향상

    [AWS Q Developer 소개]

    • 계획 관련 질문, 기존 코드 분석, 코드 작성, 테스트, 보안 검사, 디버깅, 최적화, 코드 업그레이드 등을 지원

    [AWS Q Developer의 기능]

    • 콘솔, IDE, 모바일 앱, Slack 등 다양한 곳에서 사용할 수 있게 지원

    [AWS Q를 사용할 수 있는 매체]

    • 기존 데이터를 활용해서 질문에 대한 가이드를 제공

    [탐색, 계획 단계에서의 AWS Q 활용]

    • 코드나 주석을 생성하고 기존 코드에 대한 가이드를 제공

    [작성 단계에서의 AWS Q 활용]

    • IDE Agent 형태로 기능을 제공하여 자연어 프롬프트로 애플리케이션을 개발하는 것이 가능

    [AWS Q Developer에서 제공하는 Agent에 대한 설명]

    • 단위 테스트를 생성하고, 보안 취약점 스캔, 개선 가이드 제공

    [테스트, 보안 단계에서의 AWS Q 활용]

    • 오류 발생 시, AWS 콘솔로 진단과 해결을 지원
    • 네트워크 단계 중 어디서 문제가 생겼는지 디버깅하는 VPC Analizer를 제공

    [서비스 운영 단계에서의 AWS Q 활용]

    • AWS 계정에서 사용되고 있는 리소스와 요금에 대해 분석하고 가이드를 제공받을 수 있음

    [AWS 인프라 운영에 AWS Q를 활용 가능]

    • 서비스의 언어 버전을 업데이트할 때 AWS Q Developer Agent for code transformation을 통해 전체 프로세스를 자동화 할 수 있음
    • 코드 포팅 기능도 곧 출시 예정 (2024/05/17 기준)

    [유지관리를 위한 기술 버전 업데이트에 AWS Q를 활용]

    • 대규모의 기술 개선, 개발자의 생산성 향상, 보안 및 코드 품질 개선에 사용

    [AWS Q의 사용 사례 요약]

    • 관련 데이터에 대한 깊이 있는 지식 없이도 자연어를 사용해서 통합 질문을 할 수 있음
    • 질의응답을 통해, 문제 해결에 대한 솔루션을 제안받음

    [AWS Q를 이용했을 때 이점]

    데모: 방치된 Java 애플리케이션 현대화하기

    • 데모 시나리오 상황을 아래와 같이 가정
      • 현재 버전 지원이 종료
      • 담당자가 없는 상황
      • Java 개발 경험이 없는 상황
      • 버전 업그레이드와 별개로 구현할 기능까지 있는 상황

    [Java 버전 업그레이드하는 시나리오]

    • AWS Q의 Chat을 통해 AWS에서 Java 배포 관리 방법을 알아봄


    [AWS Q Chat 기능 시연]

    • AWS Q Business를 통해 버전업에 필요한 회사 시스템 상황이나 사용하고 있는 서비스 등을 조사


    [AWS Q Business 기능 시연]

    • AWS Q Developer Agent를 통해 Java 버전 업그레이드를 진행


    [AWS Q Developer Agent 기능 시연]

    • Chat과 Inline suggestions 기능으로 테스트 커버리지를 높이는 작업을 진행


    [AWS Q Developer in the IDE 기능 시연]

    • AWS Q Developer Agent로 코드 주석을 추가하여 API 설명을 보완


    [AWS Q Developer Agent 기능 시연]

    • 버전 업그레이드로 인해 Lambda에 발생한 문제를 해결


    [AWS Q Developer Agent 기능 시연]

    • AWS CodeCatalyst에서 AWS Q에게 이슈를 할당해서 이슈를 관리하고 가이드를 받음


    [AWS CodeCatalyst, AWS Q 기능 시연]

    • AWS Q로 예상 운영 비용을 분석하여 미리 파악


    [AWS Q 기능 시연]
